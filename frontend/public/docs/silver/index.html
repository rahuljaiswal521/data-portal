    Silver Layer — Overview¶

    The Silver layer is where raw Bronze data is transformed into trusted, canonical business entities. Multiple Bronze sources are merged into a single authoritative record for each business object (customer, policy, payment, etc.).

    What is a Silver entity?¶

    A canonical entity is the single authoritative representation of a business object in the Lakehouse. Instead of having a customer table in every source system (CRM, ERP, billing), the Silver layer merges all of them into one customer entity with:

    • A single primary identity (business key, e.g. customer_id)
    • Attribute-level source priority — if CRM and ERP both have a customer name, you decide which one wins
    • Full SCD2 history — every change across all sources is tracked
    • 3NF design — no redundancy; related concepts are separate entities linked by foreign keys

    Silver schema naming¶

    Silver entities live in domain-specific schemas:

    Domain Schema Example tables
    Customer dev.slv_customer customer, customer_contact
    Policy dev.slv_policy policy, policy_coverage
    Payment dev.slv_payment payment, payment_method
    Interaction dev.slv_interaction customer_interaction, event
    Cross-domain dev.slv_xref Bridge tables, cross-reference mappings

    What you can do in the Silver layer¶

    Action Where Time
    View all entities Silver Dashboard Instant
    Design a new entity with AI Model Advisor ~5 min
    Create an entity manually New Entity wizard ~30 min
    Deploy entity to Databricks Deploy ~2 min
    Visualise entity relationships Entity Diagram Instant

    Silver Dashboard¶

    Navigate to Silver Layer → Dashboard (/silver).

    The dashboard lists all configured Silver entities with:

    Column Description
    Name Entity identifier (click to open detail)
    Domain Business domain (customer / policy / payment / etc.)
    Load type scd2 (track history) or append (insert-only)
    Sources Number of Bronze sources feeding this entity
    Status Enabled / Disabled

    How Silver entities are stored¶

    Like Bronze sources, each Silver entity generates a YAML configuration file at:

    silver_framework/conf/entities/<entity_name>.yaml

    This file defines the entity schema, its Bronze source mappings, merge logic, and attribute priority. The portal writes and manages this file for you. You can view the raw YAML on the entity detail page.

    Traditional approach (without the portal)¶

    Time saved: ~1 day per entity

    Step Without portal With portal
    Design 3NF entity schema 2–4 hrs (whiteboard + documentation) AI Model Advisor in 5 min
    Write entity YAML config 1–2 hrs (40+ fields, understanding merge logic) Generated by entity wizard
    Configure multi-source mappings 1–2 hrs (attribute priority, key resolution) Wizard step — dropdown selections
    Write Silver notebook (PySpark merge) Half day Automated by framework
    Deploy to Databricks 30–45 min One-click
    Total ~1–1.5 days ~30–40 minutes
